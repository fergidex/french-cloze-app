# French Cloze App вЂ” Project Guide

A mobile-first Next.js PWA for advanced French cloze learning, targeting B1 and B2 CEFR levels. Uses the SM-2 spaced-repetition algorithm with all progress stored in `localStorage`.

---

## Tech Stack

| Layer | Choice |
|-------|--------|
| Framework | Next.js 14 (App Router) |
| Language | TypeScript (strict) |
| Styling | Tailwind CSS v3, dark mode via `class` strategy |
| PWA | `@ducanh2912/next-pwa` (Workbox-based service worker) |
| State | React `useState` / `useEffect` вЂ” no external state library |
| Persistence | `localStorage` only вЂ” no backend, no auth |

## Core Concepts

### Sentence Format (`src/data/sentences.ts`)

Each `Sentence` object has:

```ts
{
  id: number;
  text: string;        // Sentence with ___ as the blank placeholder
  answer: string;      // The single missing word (correct answer)
  level: 'B1' | 'B2';
  category: GrammarCategory;
  explanation: string; // Grammar explanation вЂ” NOT displayed yet (future feature)
}
```

**Grammar categories** (12 total, defined in `src/types/index.ts`):
`subjunctive`, `conditional_present`, `conditional_past`, `imparfait`, `passe_compose`, `imparfait_passe_compose`, `relative_clauses`, `futur_anterieur`, `passive_voice`, `reported_speech`, `gerund`, `si_clauses`

### Cloze Mechanic

- The sentence `text` field contains `___` (three underscores) as the blank.
- `ClozeCard` splits on `___` and renders an `<input>` inline.
- **Real-time feedback**: on every keystroke, the typed value is compared character-by-character against the answer:
  - If the typed prefix **doesn't match** the answer prefix в†’ **red** input
  - If the typed prefix **matches** the answer prefix but isn't complete в†’ neutral
  - If the full word **matches exactly** в†’ **green** and `onCorrect()` fires
- Comparison is case-insensitive. Accents are required (strict mode вЂ” this is the point of the exercise).

### SM-2 Algorithm (`src/lib/sm2.ts`)

Standard SM-2 with these quality mappings:

| Button | Quality | Behaviour |
|--------|---------|-----------|
| Again | 0 | Reset: interval=1, reps=0, EF penalised heavily |
| Hard | 2 | Reset: interval=1, reps=0, EF penalised |
| Good | 4 | Progress: normal interval growth, EF unchanged |
| Easy | 5 | Progress: boosted interval growth, EF increases |

- **Interval progression**: new card в†’ 1 day в†’ 6 days в†’ `prev Г— EF` days
- **Ease Factor** starts at 2.5, minimum 1.3
- A card is **mastered** when `interval >= 21` and `repetitions >= 3`
- Again/Hard cards are **re-queued at the end** of the current session

### localStorage Schema

One key: `"french-cloze-progress"` в†’ `Record<sentenceId, CardProgress>`

```ts
interface CardProgress {
  sentenceId: number;
  interval: number;       // days
  repetitions: number;    // consecutive correct answers
  easeFactor: number;     // starts at 2.5
  nextReview: string;     // "YYYY-MM-DD"
  lastReview: string | null;
  totalCorrect: number;
  totalIncorrect: number;
}
```

## PWA / iOS Installation

1. Run `node scripts/generate-icons.js` to generate PNG icons (first-time setup).
2. Build the app: `npm run build && npm start`
3. Open in Safari on iPhone в†’ Share в†’ "Add to Home Screen"
4. The service worker (generated by next-pwa/Workbox) caches all static assets for offline use.

---

## Development

```bash
npm install
node scripts/generate-icons.js   # one-time icon generation
npm run dev                       # http://localhost:3000
```

> Note: The PWA service worker is disabled in development (`NODE_ENV=development`). Build and serve for full PWA testing.
